Why MCP Gateway

Enterprise AI Needs a Control Point

As AI agents proliferate across your organization, every tool call is an API call that touches real systems. Without a gateway, you have no visibility, no policy enforcement, and no way to audit what your agents are doing.

🔒

Zero-Trust Security

Every tool invocation is authenticated and authorized. Identity-aware filtering ensures agents only access the tools they're entitled to, with per-tool OAuth 2.0 and fine-grained RBAC powered by Authorino.

📊

Complete Observability

Per-tool metrics, structured audit logs, and traceability across every agent interaction. Know exactly which agent called which tool, when, and what happened — across your entire MCP landscape.

⚙️

Operational Control

Traffic management, guardrails, and policy enforcement at the tool level. Prevent runaway agents, protect downstream services, and maintain SLAs without burdening individual MCP server developers. Add rate limiting and token metrics with Red Hat Connectivity Link.

Capabilities

What the MCP Gateway Delivers

Built on proven Envoy proxy technology and Kubernetes-native patterns, the MCP Gateway provides enterprise capabilities at the individual tool level.

Runtime

Virtual MCP Servers

Create multiple virtual MCP servers, each aggregating a curated set of backend tools behind its own endpoint. Scope tool access by team, project, or use case — each with full isolation and consistent naming via tool prefixes.

  • Multiple isolated /mcp endpoints per gateway RHOAI
  • Each virtual server aggregates its own set of backends RHOAI
  • Tool prefix namespacing prevents collisions RHOAI
  • Streaming HTTP transport (SSE/JSON-RPC) RHOAI
Security

Identity-Aware Tool Filtering

Control which tools each agent or user can see and invoke. Allow/deny lists per tool ensure that sensitive operations are only accessible to authorized identities.

  • Per-tool authentication via OAuth 2.0 RHOAI
  • Fine-grained authorization with Authorino RHOAI
  • Tool-level allow/deny lists RHOAI
  • OAuth 2.0 Protected Resource metadata RHOAI
Observability

Per-Tool Metrics & Audit

Every tool invocation generates structured metrics and audit records. Track usage patterns, detect anomalies, and maintain compliance with full traceability across your MCP landscape.

  • Per-tool invocation metrics RHOAI
  • Structured audit logging RHOAI
  • Agent-to-tool correlation RHOAI
  • Token metrics RHCL
  • Integration with existing monitoring stacks RHOAI
Governance

Policy Enforcement at Scale

Apply traffic policies, access controls, and guardrails centrally. Protect downstream services from runaway agents and enforce organizational policies without modifying individual MCP servers. Upgrade to RHCL for rate limiting and token metrics.

  • Per-tool rate limiting RHCL
  • TLS termination and re-encryption RHOAI
  • DNS management via cluster ingress RHCL
  • Cross-namespace routing via Gateway API RHOAI
How It Works

From Agent Request to Governed Response

The gateway transparently mediates every tool call. Your agents connect to one endpoint — the gateway handles routing, security, and observability behind the scenes.

Observe
