Ecosystem Overview

The MCP Gateway sits within a larger governance ecosystem connecting discovery, registration, deployment, and runtime use of MCP servers.

Key Principle: Flow is always Catalog → Registry, never reverse. The Registry is the authoritative governance layer.

Today vs. Target

Today (3.4)

  1. Discover in MCP Catalog
  2. Deploy via Lifecycle Operator
  3. MANUALLY register in MCP Gateway
  4. MANUALLY update ConfigMap
  5. Consume in Gen AI Studio

Target (3.5+)

  1. Discover in MCP Catalog
  2. Deploy via Lifecycle Operator
  3. AUTO register in MCP Registry
  4. AUTO configure MCP Gateway
  5. Consume in Gen AI Studio

Summit Demo Workflow (May 2026)

  1. Catalog — Discover OpenShift MCP server in RHOAI catalog
  2. Deploy — Deploy via Lifecycle Operator (MCPServer CRD)
  3. Gateway Register — Manually register server with MCP Gateway
  4. ConfigMap — Update gateway configuration via ConfigMap
  5. Gen AI Studio — Discover tools in Studio tool catalog
  6. Tool Execution — Invoke tool through MCP Gateway
Governance Gap: The Registry fills the governance gap between discovery and runtime. Today's manual registration in the Gateway (step 3) will be replaced by automated registry→gateway flow in GA (Q4 2026).

Component Responsibilities

Component Responsibility
MCP Catalog Discovery, browsing, selection today; planned scope extends to Tools/Resources/Prompts/Workloads and external registry federation (see MCP Catalog)
MCP Registry Governance, lifecycle, metadata, trust
Lifecycle Operator Deployment (MCPServer CRD → Deployment + Service)
MCP Gateway Runtime connectivity and enforcement
Gen AI Studio Consumption (agent/user interaction)

RHOAI Platform Components Newly Involved

Per RHAISTRAT-1937, two additional RHOAI platform components are newly involved as the MCP Gateway Operator's external-dependency documentation strategy takes shape:

  • odh-dashboard: Agentic Starter Kit tutorials need a reference to the MCP Gateway Operator prerequisite.
  • rhods-operator: New status condition surfaced when MCP Gateway Operator CRDs are absent but MCP-dependent features are enabled.
