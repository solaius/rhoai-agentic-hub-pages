MCP Catalog

The storefront for MCP discovery. The governance and management layer of the MCP ecosystem within RHOAI.

Purpose

The Catalog provides:

  • Discovery and browsing of MCP servers
  • Selection interface for deployment
  • Publisher verification and moderation
  • Trust tier display (certified, community, unverified)
Relationship: The Catalog builds on the Registry (authoritative source), complements the Gateway (runtime enforcement), and operates in a federated and interoperable ecosystem.
Planned scope (PM vision, not shipped): The Catalog's long-term vision extends beyond MCP servers to manage Tools, Resources, Prompts, and Workloads, including workload-to-tool mappings, as first-class catalog entities. Planned capabilities include Tool Masking, Input Simplification, and Output Filtering (splitting multi-purpose tools into single-purpose variants and trimming LLM-irrelevant response fields), plus federation with external registries: Docker MCP Catalog, GitHub MCP Registry, Smithery, Microsoft Copilot Catalog, and Databricks Unity Catalog. None of this is shipped; current scope is MCP server discovery, browsing, and selection (see below).

Partner Pipeline

  1. Curation — Invitation-only selection (strategic categories)
  2. BDM Outreach — Business Development Manager outreach with consent letter
  3. Consent Letter — Partner signs consent form
  4. Technical Attestation — Ecosystem Engineering validates requirements
  5. Image Build — Build and push to quay.io/rhoai-partner-mcp
  6. Catalog Integration — Integration and testing by RH AI team

Technical Requirements for Partner MCPs

Requirement Details
Transport Streamable HTTP only (no stdio)
Deployment On-cluster deployment, local hosting only for DP; remote-hosted MCP server support planned after the DP release
Container Image UBI-based container image in Quay
Core Metadata name, description, provider, logo, license (SPDX), licenseLink, readme
MCP-Specific Metadata tools[], transportType, deploymentMode, artifacts[], endpoints[]
Security Indicators verifiedSource, sast, secureEndpoint, readOnlyTools
Access & Revocation Metadata Each tool entry also carries an access-type (read/write) flag plus revoked (boolean) and revokedReason (string) fields
