MCP Lifecycle Operator

The deployment primitive for MCP server workloads. Kubernetes operator that creates Deployment + Service from MCPServer CRD definitions.

Repository: kubernetes-sigs/mcp-lifecycle-operator
Current Version: v0.2.0 (released June 17, 2026)

Purpose

The Lifecycle Operator is the deployment layer — it deploys MCP services. The Registry represents the governed MCP and its deployment/runtime associations. The operator does NOT own governance or lifecycle state.

Component Flow

MCPServer CRD
Custom Resource Definition
→
Controller
Watches CRDs
→
Deployment + Service
Running MCP server

Configuration

The MCPServer CRD specifies:

  • Container images for MCP servers
  • Port configuration
  • Secret references for credentials
  • Environment variables
  • Resource limits

Distribution & Entitlement

Question Answer
Available in OperatorHub? No
Available via OLM? No — container image not in public OLM catalog
Non-RHOAI customers? No — requires RHOAI restricted use entitlement for OpenShift
Minimum subscription RHOAI restricted use entitlement
Installation Behavior: Does NOT deploy any servers by default. Presents a catalog during setup; user selects which to deploy. Additional servers installable as Day 2 operations via catalog UI or CRDs.
Independence from Gateway: Standalone from MCP Gateway — lifecycle operator installs first, gateway is downstream. No dependency on MCP Gateway for deployment.

Update Delivery

Lifecycle Operator updates are bundled within RHOAI operator releases:

  • Connected clusters: Automatic via RHOAI operator updates
  • Disconnected clusters: Via updated RHOAI offline bundles
  • Engineering builds new versions → hands off to RHOAI team

Qualification Responsibilities

  • OCP Team: Functional qualification (deploy, discover, manage MCP servers)
  • RHOAI Team: Deployment and integration qualification (rollout via RHOAI operator, prerequisites)
