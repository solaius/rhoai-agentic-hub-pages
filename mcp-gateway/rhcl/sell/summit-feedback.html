Summit Feedback

Customer feedback from Red Hat Summit 2026 (May 11-14, 2026) — what customers asked for and what they're evaluating.

Top 3 Customer Asks

1

Identity / Token Exchange

Customers consistently asked about identity propagation and token exchange for agent-to-tool communication. They need agents to carry user identity through to backend systems (Jira, GitHub, Salesforce) without exposing raw user credentials. This maps directly to the MCP Gateway's credential brokering capability via AuthBridge.

2

Observability

Customers need full visibility into agent-to-tool invocations — who called which tool, when, with what parameters, and what the result was. This includes metrics, traces, and audit logs with agent-to-tool correlation. This maps to the MCP Gateway's OpenTelemetry tracing and structured audit logging.

3

Cost Optimization / Context Bloat

Customers are concerned about token costs and context bloat as agents invoke more tools. They need per-tool token metrics, cost attribution, and rate limiting to prevent runaway usage. This maps to the MCP Gateway's token metrics and Limitador rate limiting.

Note: Agent-to-Agent (A2A) communication was not yet a major ask at Summit. The focus was on agent-to-tool governance, not agent orchestration. A2A is on the roadmap but is not the primary customer pain point today.

Common Thread

Insight: Customers want MCP/tool governance solutions, not just a gateway.

The recurring theme across Summit conversations: customers are looking for enterprise-grade governance for agent-to-tool communication. They need:

  • Identity propagation — agents carry user identity to backend systems
  • Access control — agents see only the tools they're authorized for
  • Observability — full visibility into agent-to-tool invocations
  • Cost control — token metrics and rate limiting to prevent runaway usage
  • Multi-tenancy — different teams/groups see different tool catalogs
  • Centralized management — single control plane for all MCP servers

This is not a traditional API gateway use case. This is a new category: agent governance.

What This Means for Positioning

The MCP Gateway is not "just a gateway" — it's an agent governance platform. The value proposition is not about routing efficiency or protocol translation. It's about enabling enterprises to deploy agents safely at scale with full governance and observability.

Key messaging takeaways:

  • Lead with identity/token exchange — this is the #1 customer ask
  • Emphasize observability as a first-class capability, not an add-on
  • Position cost control as essential for production deployments
  • Frame A2A as future state, not current priority
  • Tie everything back to governance — the unifying theme
