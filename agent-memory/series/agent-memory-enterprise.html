Healthcare

Clinicians spend more time on documentation than care. Agent memory enables longitudinal patient context across encounters — the agent that triaged in February references that context in June.

SOX • SEC 17a-4 • GDPR/SOX erasure collision

REGULATORY LANDSCAPE

The Compliance Timeline

Three regulatory frameworks are converging in 2026 to make governed memory a deployment prerequisite, not a feature.

August 2, 2026

EU AI Act

  • Tamper-evident logging, 6-month minimum
  • Human oversight mechanisms
  • Multi-agent chain compliance (Recitals 99-100)
  • Up to €20M or 4% of turnover for non-compliance
2026 Update

HIPAA Security Rule

  • Eliminates “addressable” safeguard distinction
  • AI systems explicitly in annual risk assessments
  • Vector DB provenance mapping for PHI
  • BAAs required for AI with PHI access
2026 Report

FINRA Supervisory Risk

  • AI agents classified as distinct supervisory risk
  • 7-year audit trail, individual attribution
  • Complete reasoning chain preservation
  • GDPR/SOX erasure collision
⚠
The GDPR / Retention Collision

GDPR Article 17 says delete personal data on request. SOX and SEC say retain for 7 years. EU AI Act says keep logs for 6 months minimum. For memory systems, this collision is a data architecture problem: selective erasure of personal identifiers while preserving the audit-relevant decision chain. No team-level implementation can solve this. It requires platform-level record metadata, provenance tracking, and policy-driven lifecycle management.

THE PLATFORM ARGUMENT

What a Platform Standardizes

The convergence table reveals what should be built once at the platform level versus what each vertical configures.

Platform-Standardized

The same across every vertical:

  • Governed memory substrate with 4 access patterns
  • Multi-tier scope isolation with RBAC
  • Append-only, immutable audit trail
  • Inspectable compaction
  • Write-path governance hooks
  • Memory-over-MCP through a gateway
Vertical-Customized

Configured per deployment, not built per vertical:

  • Scope tier definitions (patient vs. client vs. classification)
  • Retention policies (6yr HIPAA / 7yr SOX / 30yr equipment)
  • Erasure semantics (GDPR / SOX-trumps / records-authority)
  • Compaction policies (what to preserve vs. compress)
  • Deployment mode (cloud / on-prem / edge / air-gapped)
  • Write-path approval workflows
