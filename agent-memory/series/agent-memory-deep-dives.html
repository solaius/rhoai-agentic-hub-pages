Seven Cross-Harness Patterns
1. Filesystem-Native Markdown

Universal pattern. All five harnesses use Markdown files as primary memory format. “The files are the agent.”

2. Layered Override Semantics

Global → project → workspace → session. More specific scopes override broader ones. Policy separate from memory.

3. Memory-as-MCP

OpenClaw and Hermes expose memory through MCP servers. Memory becomes a governed tool with per-tool authorization and audit.

4. Dream Consolidation

Between-session background processes that reorganize memory. Claude and Codex both implement this; OpenClaw has optional dreaming.

5. Git-Native Versioning

Filesystem storage makes version control with Git natural. Memory changes are diffable, reviewable, and revertible.

6. Policy ≠ Memory

Every harness separates human-authored configuration (CLAUDE.md, SOUL.md) from agent-written memory. Conflating them creates governance confusion.

7. Capture ≠ Consolidate

Separating the act of remembering (inline, during session) from organizing (async, between sessions) is the core architectural insight.

MEMORY CONSOLIDATION

How “Dreaming” Works

Sleep-time consolidation — running memory reorganization between sessions — is not a research curiosity. It’s a validated production pattern. Without it, agents degenerate within 48 simulated hours.

The Converging Memory Layer Model

All major harnesses independently arrived at a similar layered structure. The layer names differ; the separation of concerns does not.

Layer 1
Policy / Configuration — Human-authored instructions. Highest authority. Not memory — it’s configuration.
CLAUDE.md • SOUL.md • AGENTS.md • hermes.md • GEMINI.md
Layer 2
Persistent Memory — Agent self-written knowledge that persists across sessions. The “pen” — captures what’s worth remembering.
MEMORY.md • USER.md • topic files • daily logs
Layer 3
Session Context — Ephemeral conversation state. High-value items promoted to Layer 2. Lost when session ends.
Layer 4
Consolidation — Background process that reorganizes Layer 2 between sessions. The “editor” — prunes, merges, resolves contradictions.
Claude Auto Dream • Codex Memories pipeline • Letta sleep-time compute

The Governance Gate

Claude Dreaming and Anthropic’s Managed Agents both require human approval before reorganized memory is deployed. This prevents autonomous self-modification of organizational memory. The pattern: propose → review → approve/reject → deploy.

ADVERSARIAL MEMORY

Memory Security Is Existential

Stateless LLMs have bounded threat surfaces — the attack ends when the context window closes. Memory makes attacks persistent, propagating, and implicitly trusted. OWASP classified this as ASI06 in the 2026 Agentic AI Top 10.

NeurIPS 2025

MINJA Attack

>95%
injection success rate

Injects malicious records via query-only interaction — no elevated privileges needed. Uses “bridging steps” and temporal decoupling. Existing defenses fail because they detect malicious actions, not corrupted beliefs.

Cascade Risk

Memory Contamination

87%
downstream decisions poisoned

One compromised agent’s memory propagates to 87% of downstream agents within 4 hours in multi-agent systems. No external validation mechanism exists — agents trust their own memories implicitly.

December 2025

MemoryGraft & SpAIware

MemoryGraft implants malicious “successful experiences” into long-term memory — structurally indistinguishable from legitimate context. SpAIware persists across sessions through document injection.

The Benchmark-to-Production Gap

Headline scores (92-96%) come from moderate-scale benchmarks. At 10M tokens — roughly a year of daily agent conversations — the best system scores 64.1%. No benchmark tests compliance (GDPR erasure, HIPAA isolation), multi-tenant isolation, security (injection resistance), or production reliability under scale degradation. These are the dimensions that matter most for enterprise deployments.

KV-CACHE OPTIMIZATION

Why Memory Ordering Affects Inference Cost

The order in which memory items are assembled into the context window directly determines KV-cache hit rates. This is a cross-layer optimization that only a platform vendor with control over both the memory service and the inference engine can deliver.

The Problem

If 100 concurrent agents receive memory in a different order every request, each pays full prefill cost. KV-cache can’t reuse any previous computation because the prefix has changed.

Cache miss → full recomputation → 2,850ms TTFT
The Solution

If memory is returned in a deterministic, stable order — static prefixes before dynamic content — all agents sharing the same memory tier pay prefill cost only once.

Cache hit → reuse KV pairs → 340ms TTFT (8.4x faster)

Ordering Principle

High-stability, high-weight memories first (enterprise/org scope — shared by the most agents, change infrequently, maximum cache reuse). Dynamic, session-specific content last. Variable content (timestamps, session IDs) at the very end.

MULTI-MODAL MEMORY

Beyond Text: A 12–24 Month Horizon

Multi-modal input processing (a model that sees an image) is not multi-modal memory (an agent that remembers what it saw). The distinction matters for architecture.

Today

Multi-Modal Processing

Model accepts images/audio/video as transient inputs. Reasons over them in context. Does not retain them after the session ends. The model “sees” but does not “remember.”

Emerging

Multi-Modal Memory

Agent persists, organizes, and retrieves non-text experiences across sessions. Recalls a chart from a meeting, a frame from a scan. Requires encoding, storage, cross-modal retrieval, and consolidation.

Near-Term
Convert-to-Text Bridge

Caption images, transcribe audio, extract text from documents. Store the text representation. Low cost, works today.

Medium-Term
Multi-Modal Embeddings

Unified embedding space (CLIP, ImageBind, Gemini Embedding 2). Cross-modal retrieval. Storage asymmetry challenges.

Longer-Term
Full Multi-Modal Providers

Native multi-modal memory providers. OmniMem reported +411% F1 improvement. Schema extensibility (source_modality fields) should be baked in now.
