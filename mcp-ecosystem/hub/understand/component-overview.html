MCP Gateway

MCP Gateway
v0.7.1 TP

Purpose: API gateway for AI tool calls. Provides virtual MCP servers, identity-aware tool filtering, per-tool authentication, metrics, and audit logging.

Status: Technical Preview (TP) as of RHCL 1.3.3 (April 30, 2026). Latest release v0.7.1 (June 18, 2026). GA targeted for Q4 2026 with MCP Registry integration.

Key CRDs:

  • MCPServer — Registers an MCP server with the gateway (toolPrefix, targetRef to HTTPRoute)
  • MCPGatewayExtension — Configures gateway-level behavior (trustedHeadersKey, broker settings)

Upstream Repo: Kuadrant/mcp-gateway

Platform: Built on Red Hat Connectivity Link (RHCL). RHCL is a WASM plugin for Envoy (not a standalone gateway), meaning minimal disruption to existing infrastructure.

RHOAI Integration: Ships as part of RHOAI subscription (auth via Authorino, rate limiting for AI workloads only). Full RHCL purchase required for DNS, HA/DR/GLB.

Roadmap Highlights:

  • TP (Apr 2026): Isolated Gateway Deployment, Vault Integration, MCP Elicitation, Content Guardrails, OTEL Tracing
  • GA (Q4 2026): Deployment Operator, Security Plugins (DDoS, Bot Management), Session Management, HA with health checks, MCP Registry integration

MCP Catalog

MCP Catalog
3.4 DP

Purpose: Discovery layer for MCP servers. Browse partner, community, and Red Hat MCP servers with metadata (tool lists, security indicators, license, provider logo, README).

Status: Developer Preview (DP) shipped in RHOAI 3.4 (code freeze April 10, 2026). TP in 3.5 (Aug 2026) with Registry integration. GA targeted for Q4 2026.

What Shipped (3.4 DP):

  • 5 Partners: Confluent, Dynatrace, Terraform (HashiCorp), Microsoft Azure, EnterpriseDB (EDB)
  • 2 Community: MariaDB, MongoDB
  • 3 Red Hat: OpenShift, Ansible AAP, + 1 other
  • Quay org: quay.io/rhoai-partner-mcp
  • Upstream: opendatahub-io/model-metadata-collection (PR merged)

Technical Requirements for MCP Servers:

  • Streamable HTTP transport only (no stdio)
  • On-cluster deployment, local hosting only (no remote for DP)
  • UBI-based container image in Quay
  • Core metadata: name, description, provider, logo, license (SPDX), licenseLink, readme
  • MCP-specific: tools[], transportType, deploymentMode, artifacts[], endpoints[]
  • Security indicators: verifiedSource, sast, secureEndpoint, readOnlyTools

Key Features:

  • Trust tier display (certified, community, unverified) — sourced from Registry in 3.5+
  • RBAC and approval workflow integration with Registry
  • Multi-registry federation support (Docker MCP Catalog, GitHub MCP Registry, partner catalogs)

Current Limitation: Catalog does NOT create prerequisites automatically (ServiceAccount, ConfigMap, ClusterRoleBinding must be created manually before deploying from catalog in 3.4).

3.5+ Improvement: Registry integration enables automatic prerequisite creation via Lifecycle Operator.

MCP Lifecycle Operator

MCP Lifecycle Operator (MCPLO)
v0.2.0 DP

Purpose: Kubernetes operator for deploying and managing MCP server workloads. Creates Deployment + Service from MCPServer CRD definitions.

Status: Developer Preview (DP) as of v0.2.0 (June 17, 2026). TP in RHOAI 3.5 (Aug 2026). GA targeted for Q4 2026.

Key CRD:

  • MCPServer — Defines container image, ports, secrets, env vars, resource limits. Operator creates Deployment + Service + prerequisites.

Upstream Repo: kubernetes-sigs/mcp-lifecycle-operator

Distribution & Entitlement:

  • NOT available in OperatorHub or via OLM
  • Container image not in public OLM catalog
  • Requires RHOAI restricted use entitlement for OpenShift subscription minimum
  • Bundled within RHOAI operator releases
  • Disconnected installation only via RHOAI offline bundle

Installation Behavior:

  • Does NOT deploy any MCP servers by default upon installation
  • Presents catalog of available MCP servers during setup (e.g., OCP MCP, ACM MCP)
  • User selects which servers to deploy during initial wizard
  • Additional servers installable as Day 2 operations via catalog UI or CRDs
  • No dependency on MCP Gateway — lifecycle operator installs first; gateway is downstream

Supported MCP Servers:

  • OpenShift (Available)
  • Ansible AAP (Available)
  • Red Hat Satellite (WIP)
  • Red Hat Insights (Available)
  • Linux/RHEL (WIP)

Update Delivery:

  • Engineering builds new versions → hands off to RHOAI team
  • RHOAI bundles updates in RHOAI operator releases
  • Connected clusters: automatic via RHOAI operator updates
  • Disconnected clusters: via updated RHOAI offline bundles

MCP Registry

MCP Registry
Design Phase

Purpose: Authoritative governance layer for MCP servers. System of record that brings governance, lifecycle control, trust, and visibility so MCP assets can move cleanly from discovery through approval, deployment, runtime use, and consumption.

Status: Design phase. Dev Preview (DP) targeted for RHOAI 3.5 (Aug 2026). GA targeted for Q4 2026.

What It Solves:

In 3.4, the MCP ecosystem is fragmented and manual. The Registry fills the governance gap connecting Catalog (discovery) → Deployment (MCPLO) → Runtime (Gateway) → Consumption (Studio).

Core Capabilities (MVP):

  • Governed registration of MCP servers
  • Metadata-first MCP records (follows upstream MCP registry schema)
  • Lifecycle state tracking (Draft → Candidate → Verified → Approved → Published → Deprecated → Retired)
  • Certification and approval visibility
  • Support for internal and external MCP representation
  • RBAC-aware visibility and governance
  • Conceptual integration with catalog, deployment, gateway, AAA/Studio

Proposed Data Model (Not Finalized):

Source: MLflow MCP Registry Data Model Proposal — this is a proposal/brain dump, not an approved design.

Entity Role Key Properties
MCPServer Logical governed asset name, description, status, tags, aliases, workspace
MCPServerVersion Immutable MCP payload + mutable governance metadata version, server_json (immutable), lifecycle/approval/verification/certification states, tags, source, run_id, is_deployed

Governance Tracks (Proposed — Not Linear Lifecycle):

Track States Purpose
Lifecycle State Draft, Candidate, Published, Deprecated, Retired Progression state of the version
Approval Status Draft, Pending, Approved, Rejected, Revoked Authorization/governance decision
Verification Status Unverified, Verified Identity/source verification
Certification Status None, Candidate, Certified, Expired, Revoked Certification program compliance

Integration Points:

  • Catalog: Servers from the Catalog are registered INTO the Registry for governance tracking; Registry's certification status may inform trust tier labels displayed in the Catalog UI
  • MCPLO: Reports deployment state to Registry (is_deployed flag)
  • Gateway: Polls Registry for approved MCP servers; auto-creates MCPServer CRDs for runtime enforcement
  • Gen AI Studio: Queries Registry API for approved MCP servers (RBAC-aware, no manual ConfigMap)
