What Is MCP Management?

MCP Management is Red Hat's unified ecosystem for discovering, governing, deploying, and consuming Model Context Protocol servers at enterprise scale.

The Problem: MCP Without Governance

The Model Context Protocol (MCP) is an open standard for connecting AI agents to external tools and data sources. It defines how agents discover available tools, understand what each tool does, and invoke those tools using a common JSON-RPC protocol.

What is MCP? — Protocol Basics

An MCP server exposes one or more tools via the MCP protocol. For example, a Jira MCP server exposes create_issue, search_issues, update_issue tools. An OpenShift MCP server exposes tools for querying cluster state, managing workloads, and troubleshooting deployments.

Key MCP operations:

  • tools/list — Discover what tools a server offers
  • tools/call — Invoke a specific tool with parameters
  • initialize — Establish a connection to the server

Read the full MCP specification →

But MCP servers in production are not simple. They require:

  • Discovery — How do users find the right MCP server for their needs?
  • Governance — Who approved this MCP server? Is it certified? Who owns it?
  • Deployment — How do I deploy it to Kubernetes? What RBAC does it need?
  • Security — How do I enforce "Agent A can only call these tools"?
  • Integration — How does Gen AI Studio know this MCP server exists?

The MCP Management Journey

RHOAI 3.4 marked the start of the MCP Management Ecosystem — the first release where MCP components shipped as integrated products. Each subsequent release builds out more of the end-to-end governed workflow.

Pre-3.4: Fully Manual MCP
  1. Find — Search GitHub for MCP servers, hope they're containerized
  2. Deploy — Write your own Deployment YAML, figure out RBAC, configure manually
  3. Register — No standard registration mechanism
  4. Secure — Roll your own auth, no per-tool policies
  5. Consume — Wire up manually to your agent framework

No catalog, no operator, no governance, no gateway. Every step required deep Kubernetes expertise and custom scripting.

RHOAI 3.4: MCP Management Begins DP
  1. Discover — Browse the MCP Catalog in AI Hub (10 servers: 3 Red Hat [OpenShift, Ansible AAP, Red Hat Insights/Lightspeed], 5 partner, 2 community)
  2. Deploy — Deploy from catalog via MCP Lifecycle Operator DP (prerequisites still manual)
  3. Register — Manually edit gen-ai-aa-mcp-servers ConfigMap to make servers visible in Gen AI Studio
  4. Secure — MCP Gateway TP via RHCL provides per-tool auth, identity-aware filtering
  5. Consume — Use MCP tools from Gen AI Studio playground

First integrated experience. Catalog + Operator automate discovery and deployment. Gateway adds enterprise security. ConfigMap bridge is manual but functional.

RHOAI 3.5+: Full End-to-End MCP Management Target
  1. Discover — Catalog with trust tier labels (certified, community, unverified)
  2. Deploy — One-click deploy with automated prerequisite creation
  3. Register — MCP Registry DP captures governance metadata, replaces ConfigMap bridge
  4. Secure — Gateway auto-discovers registered servers, enforces per-tool policies
  5. Consume — Gen AI Studio reads from Registry, showing only approved MCP servers

Governed end-to-end. Full audit trail. RBAC-aware. Approval workflows for production deployments. Registry-to-Gateway automation.

What Is the MCP Management Ecosystem?

MCP Management is the end-to-end product experience for working with MCP servers in Red Hat OpenShift AI. It brings together four components:

MCP Catalog
DP

Discovery layer. Browse partner, community, and Red Hat MCP servers. View tool lists, security indicators, and deployment requirements. Filter by trust tier and license.

MCP Registry
DP (3.5)

Governance layer. Currently in design phase; Dev Preview (DP) targeted for RHOAI 3.5. System of record for MCP servers, tracking server status, version publish-state, ownership, and deployment visibility.

MCP Lifecycle Operator
DP

Deployment primitive. Kubernetes operator that deploys MCP server workloads. Creates Deployment, Service, prerequisites (ServiceAccount, ConfigMap) from MCPServer CRD.

MCP Gateway
TP

Runtime enforcement. API gateway for AI tool calls. Provides identity-aware tool filtering, per-tool authentication, virtual MCP servers, metrics, and audit logging.

Position Statement

MCP Management is the enterprise governance layer for AI tool access. It brings the same rigor to MCP servers that Red Hat already provides for container images (Quay), models (Model Registry), and APIs (3scale) — with discovery catalogs, governed registries, automated deployment, and runtime policy enforcement.

This is not just tooling for MCP. It's the foundation for governing all AI assets (agents, skills, guardrails) in production.
