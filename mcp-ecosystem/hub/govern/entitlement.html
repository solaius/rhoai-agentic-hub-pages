Subscription Tiers

OpenShift Container Platform (Base)

Includes MCP Gateway (basic) for platform-internal MCP servers:

  • MCP Gateway proxy functionality (broker, router, controller)
  • No authentication, rate limiting, or observability
  • For platform-internal use cases only

RHOAI restricted use entitlement

Minimal AI capabilities for MCP server deployment:

  • MCP Lifecycle Operator (deployment primitive)
  • MCP Catalog (discovery and deployment UI)
  • MCP Gateway (basic proxy, no auth/rate limiting)
  • Not available: OperatorHub, OLM, standalone OCP installation
  • Distribution: Bundled in RHOAI operator only; offline/disconnected via RHOAI bundle

RHOAI (Full)

Complete AI platform with MCP governance and enforcement:

  • All RHOAI restricted use entitlement components
  • MCP Registry — governance, lifecycle, trust (DP planned, not landing in RHOAI 3.5 stable; TP targeted RHOAI 3.6 EA1, PM direction Jul 2026; GA Q4 2026)
  • Authorino — authentication and policy enforcement
  • Limitador — rate limiting (AI workloads only, not MCP/Agents)
  • Gen AI Studio MCP Support — tool catalog, playground integration
Key Limitation: RHOAI rate limiting and token metrics apply to MaaS workloads only. For MCP and Agent rate limiting, Full RHCL subscription is required.
Same caveat applies: if the AI BU MaaS-parity direction is confirmed (see the Token Scope Limitation callout above), MCP and Agent rate limiting would also move out of the Full-RHCL-only scope stated here. Not yet confirmed.

Full RHCL

Enterprise-grade connectivity platform with multi-cluster resiliency:

  • All RHOAI components
  • DNS management — automatic failover, geo-routing
  • High Availability / Disaster Recovery — cross-cluster mirroring
  • Global Load Balancing — multi-cluster traffic distribution
  • Advanced Gateway Features — TLS reencryption, external provider egress, DDoS protection
  • MCP/Agent Rate Limiting — rate limiting for non-MaaS AI workloads
  • Single-cluster constraint: Red Hat supports only a single instance of RHCL per cluster
  • Non-AI workloads: customers using RHCL features for non-AI workloads or use cases must purchase a separate RHCL SKU

RHOAI-RHCL Support Agreement

The RHOAI-RHCL support agreement is fully executed: all 10 stakeholders across RHCL and RHAI (PM, Engineering, and Support on both sides) signed between Mar 23 and Apr 28, 2026. Full scope and terms are maintained internally. Red Hat associates: contact the MCP Gateway PM for details.

Scope: Case 1 only. This agreement covers Case 1: RHOAI deploying and using specific versions of RHCL components via images pushed to the registry by the RHCL team. Case 2, customers installing the full RHCL product alongside RHOAI for extended functionality, will be covered by a separate agreement, to be developed once those use cases are defined.
Known Gap (Aug 2026): RHOAI 3.5 (Aug 2026) supports OCP 4.19, but RHCL 1.4 likely won't. Configuration is untested, best-effort only. OCP 4.19 maintenance support ends Dec 2026 (4-month overlap). Customers advised to upgrade to OCP 4.20+ for full support.

Distribution Channels

MCP Lifecycle Operator

  • Not available in OperatorHub or via OLM
  • Not available for standalone OCP installation
  • Requires RHOAI restricted use entitlement subscription minimum
  • Distribution: Bundled within RHOAI operator releases
  • Connected clusters: Automatic via RHOAI operator updates
  • Disconnected clusters: Via updated RHOAI offline bundles

MCP Gateway

  • Basic gateway: Ships with OCP for platform-internal MCP servers
  • Auth + Rate Limiting: RHOAI subscription required (AI workloads only)
  • Full RHCL features: RHCL subscription required (DNS, HA/DR/GLB, MCP/Agent rate limiting)
